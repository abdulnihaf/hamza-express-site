# TV Mission Control

View-only cockpit for the Hamza Express 6-TV creative production pipeline.

**Live URL:** https://hamzaexpress.in/ops/tv-mission-control/

## What it shows

1. **The Wall** — physical mockup of all 6 TVs with current locked creative per screen
2. **Production Pipeline** — per-conveyance state (References → Heroes → Finals) with progress %
3. **Asset Library** — filterable grid of every asset (LOCKED / WIP / Missing / References)

## How it's updated

Assets + manifest mirrored from local creative library:

```bash
# On Nihaf's Mac:
python3 ~/Documents/HE_Creative_Library/_ui/build_for_deploy.py

# Then commit + push:
cd ~/Documents/Tech/hamza-express-site
git add ops/tv-mission-control/
git commit -m "tv-mission-control: refresh manifest + assets"
git push origin main  # auto-deploys via Cloudflare Pages
```

## Files

| Path | Purpose |
|---|---|
| `index.html` | The cockpit UI (vanilla HTML + Tailwind CDN, no build step) |
| `manifest.json` | Production state generated by build_for_deploy.py |
| `assets/01_Reference_Images/` | Item refs (Item_Ghee_Rice.jpg etc.) |
| `assets/02_Hero_Images_WIP/` | WIP heroes awaiting approval |
| `assets/02_Hero_Images_LOCKED/` | Approved hero food shots |
| `assets/03_Final_Creatives_WIP/` | Final TV creatives in progress |
| `assets/DESKTOP_FINAL/` | Production-ready finals (mirror of ~/Desktop/HE_FINAL_OUTPUTS) |

## Source of truth

The local working library lives at `~/Documents/HE_Creative_Library/` — this `ops/tv-mission-control/` folder is just a deploy mirror. Don't edit files here directly; edit in the library and re-run `build_for_deploy.py`.
