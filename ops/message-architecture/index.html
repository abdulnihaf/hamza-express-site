Hamza Express · WhatsApp Message Architecture

A surgical audit of every inbound message across 223 conversations (1,411 messages, 336 free-text, 218 taps) — mapped to a unified bot architecture that funnels every user, in every scenario, to one of four universal CTAs: Book Table, View Menu (link), Get Directions, Order Takeaway.

⚡ Core principle: The outlet is the hero. The bot's job is to get people into the outlet (dine-in) or picking up from the outlet (takeaway). View Menu = link to website. MPM (catalog) = only for the WABA takeaway checkout. Talk to Faheem = always one tap away.

01The Four Universal CTAs

Every response from the bot must end with one of these four (or a relevant subset). They are the only exits from any conversation state.

📍
1 · Get Directions
PRIMARY — outlet is the hero. CTA URL button → hamzaexpress.in/go/maps opens Google Maps pin. Text: address + landmark + hours. Static map image for preview. Auto-appended on every booking + every paid order.
📅
2 · Book Table
DINE-IN converter. Native WA Flow (nfm) — date, time, party size, name, phone. Writes to bookings. 14 flows completed. Ops sees at /ops/bookings/. Pre-fills for regulars.
📖
3 · View Menu
URL LINK — not catalog. cta_url button → hamzaexpress.in/#menu. Full menu on the website (faster, richer, has photos + 1918 story). MPM is NOT used here. Intent: let them decide, then visit or order.
🥡
4 · Order Takeaway
ONLY HERE → MPM fires. Product list (catalog) → cart → order_details UPI payment → Odoo POS → ready ping. 34 carts, 27 paid. This is the only path that uses a product_list message.
⚡ Outlet-first hierarchy: Directions ranks first, Book second, Menu third, Takeaway last. Most Bangalore customers live within 10 km — dine-in > takeaway > delivery (Swiggy/Zomato). The bot's job is not to replace the outlet but to pull people into it. Every paid takeaway and every booking auto-appends the Directions card so they can't get lost.
🛟 The Fifth Escape Hatch — Talk to Faheem: Available on every message, every scenario, every state. Shown as a subtle [💬 Talk to a person] link at the bottom of every CTA card. Fires escalateToFaheem(waId, reason, lastMessages) — pauses bot, pings Faheem's ops UI, Faheem takes over via /ops/leads/. Bot auto-resumes after 24h of Faheem silence or explicit handback.

05Intent Gaps — Each One Hurts Revenue

These are the places where users typed something real and the bot failed or fumbled. Every gap below is a concrete fix.

Delivery ask · 9 messages · ~4% of organic text

"home delivery is there" · "does it have door step delivery?" · "delivery noida sector 93" · "delivery not available" · "can you deliver to my location" · "are you able to deliver here faheem - [maps link]"
PROPOSED RESPONSE Hi! We don't do direct delivery from WhatsApp — but you can order us on Swiggy or Zomato (links below). For pickup, we're 7 min from Russell Market.

CTAs [🛵 Swiggy] [🛵 Zomato] · [🥡 Order for Pickup] · [📍 Get Directions]

BOT WIRING Intent classifier regex \b(delivery|deliver|home|doorstep|parcel)\b → handleDeliveryDeflect(waId) → send text + 4-button interactive.

COD ask · 1+ message (cod available hai kya)

"cod available hai kya", "cash on delivery"
PROPOSED RESPONSE We're a pickup-first kitchen — so no COD on direct orders. Pay UPI here, we have it ready in 15 min. For doorstep + COD, use Swiggy/Zomato.

CTAs [🛵 Swiggy] [🛵 Zomato] · [🥡 Order Pickup (UPI)]

BOT WIRING Regex \bcod\b|cash on delivery|pay cash → handleCodDeflect — same block as delivery, different copy.

Specific dish availability · 5 messages

"do you have a mandhi" · "paneer hai ya nhi" · "1 full paneer masala" · "10 roti" · "i need tea also"
PROPOSED RESPONSE Yes — Paneer Masala is available at ₹210, Roti at ₹15 each. We don't stock Mandi right now, but our closest match is Ghee Rice + Chicken Kabab. See full menu:

CTAs [📖 View Menu] · [🥡 Order Now]

BOT WIRING New handler: handleDishQuery(text) — regex-match against PRODUCTS keys + DISH_ALIASES table. If matched → reply with price + MPM. If not matched (mandi, tea-only) → honest "not available, closest is X" + MPM. This is the biggest missed conversion signal.

Cancel / refund · 14 messages

"cancel" (x13) · "cancel my order"
PROPOSED RESPONSE Got it. Your order #HE-XXXX — do you want to (a) cancel before we start cooking (auto-refund to UPI in 2 min), or (b) cancel after pickup (must talk to staff)?

CTAs [❌ Cancel (auto-refund)] · [💬 Talk to staff]

BOT WIRING Regex ^cancel\b|refund|not want → handleCancel(waId) — look up latest order, check Odoo state. If in draft: auto-cancel + Razorpay refund. If in paid+KDS: escalate to Faheem.

Location / directions (free-text) · 7 messages

"location" · "where is the location" · "which names location?" · "this is from hebbal banglore" · "banglore available?"
PROPOSED RESPONSE 📍 Hamza Express

06Master Intent Router — The Decision Tree

Every inbound message, from every entry point, goes through one function: routeInboundMessage(msg). This is its full decision table.

// Pseudocode — exhaustive routeInboundMessage(msg) { // 1. META-LEVEL CHECKS (these short-circuit early) if isHiringCandidate(msg.waId) return skip; // HN hiring filter — existing if isAdmin(msg.waId) return skip; // Nihaf, Naveen testers if msg.isStaffReply return logOnly; // Faheem typing from dashboard // 2. NON-TEXT HANDLERS (small set, total ~17 msgs/lifetime) switch (msg.type) { case "nfm_reply" : return handleBookingFlowComplete(msg); // book table flow → write bookings row case "order" : return handleCartSubmit(msg); // catalog cart → payment case "button_reply" : return handleButton(msg.buttonId); // jump to §07 case "list_reply" : return handleButton(msg.listId); // same namespace case "image" : return sendMediaUnsupported + 4CTA; case "location" : return handleSharedLocation(msg); // distance-aware case "reaction" : return logOnly; // no response — silent case "unsupported" : return sendMediaUnsupported + 4CTA; } // 3. TEXT INTENT CLASSIFIER (ordered — first match wins) text = normalize(msg.content); intent = classify(text); switch (intent) { case "meta_ad_combo" : return sendComboMPM + secondaryCTAs; // existing path case "station_qr_*" : return sendStationMPM(station); // existing path case "menu_request" : return sendFullMenuMPM + 4CTA; case "order_intent" : return sendTakeawayMPM + CTA[book,dir]; case "booking_intent" : return launchBookingFlow + CTA[menu,dir]; case "location_direction" : return sendDirectionsCard + 4CTA; case "delivery_ask" : return sendDeliveryDeflect + aggregators + takeaway; case "cod_ask" : return sendCodDeflect + aggregators + takeaway; case "specific_dish" : return dishLookup(text) → price + MPM; case "timing_ask" : return sendTimingCard + 4CTA; case "call_phone" : return sendCallCard (phone CTA) + 4CTA; case "cancel_refund" : return handleCancel(latestOrder); case "complaint" : return escalateToFaheem + apology; case "status_query" : return lookupOrderStatus + text; case "swiggy_zomato" : return sendAggregatorLinks + 4CTA; case "greeting" : return sendTierGreeting + 4CTA; case "thanks" : return reactWithHeart (no reply); case "yes_no" : return contextual — check session state; case "hiring_candidate" : return handOffToHiringBot; case "gibberish" : return send4CTAFallback; default : return send4CTAFallback + log unclassified; } }

09CTA 3 · View Menu — Deep Spec (URL LINK, not MPM)

Key decision: View Menu is always a URL button that opens hamzaexpress.in/#menu. It is never a catalog product_list. The product_list (MPM) is reserved exclusively for the Order Takeaway flow (§12). Reason: menu browsing is a discovery act — the website has photos, the 1918 story, combos grouped visually, and it's faster to scroll than scrolling a WA catalog. WA catalog friction only makes sense when they're ready to pay.

Trigger conditions

  • Button tap: see_dine_menu, meta_ad_see_menu, see_combos, view_menu
  • Intent match: menu_request ("menu", "send menu", "menu card", "what do you have")
  • Post-booking ping: "while you plan — browse the full menu"
  • Generic discovery state — user hasn't picked a CTA yet

Delivery format

  • Primary: cta_url button — "📖 View Full Menu" → hamzaexpress.in/#menu
  • Text above button: "Here's our full menu with photos, combos, and prices. Tap below 👇"
  • Secondary CTAs after: [📍 Directions] · [📅 Book Table] · [🥡 Order for Pickup]
  • Never send product_list here — that's the takeaway checkout path
  • If user taps "Order for Pickup" next → THAT's when MPM fires

14Invariants · Design Rules

Always true

  • Every response ends with at least one of the 4 CTAs
  • Max 2 outbound messages per inbound (main + CTA card)
  • Tier-adaptive — new users get more context, regulars get action-first
  • Every outbound written to wa_messages
  • Every inbound classified — intent column never null

Never do

  • Ask for name if station QR or CTWA context exists
  • Send 5th CTA (menu has exactly 4)
  • Reply to reactions (emoji on bot message)
  • Reply to spam pitches (silent flag)
  • Respond to hiring candidates (HIRING_DB filter active)

Escalate to Faheem

  • Complaint intent (contains bad/worst/cold/stale/missing)
  • 3+ unclassified in a row
  • Cancel after order is in KDS
  • Explicit "talk to staff" tap
  • Any booking with party size > 8

16Human Handoff — Universal Escape Hatch to Faheem

At every point, in every state, the user can reach a human. Three triggers, one outcome: Faheem takes over.

Explicit — user asks

  • Tap: [💬 Talk to a person] — always visible at bottom of any CTA card
  • Text: "talk to staff", "speak to someone", "human", "real person", "agent"
  • Text: "call me", "can I call", "phone number", "contact number"
  • Text: "give reply sir", "faheem", "owner", "manager"

Implicit — bot detects need

  • 3 unclassified messages in a row
  • Complaint keywords (bad, cold, stale, missing, waste, problem)
  • Cancel on paid+KDS order (can't auto-refund)
  • Booking party size > 8 (needs confirmation)
  • Dish query matching nothing (mandi, tea-only, etc)
  • Long message (>150 chars) that classifier can't parse
  • User repeats same question 2x (bot didn't answer)

Forced — system triggers

  • Odoo POS session failed (order can't be created)
  • Razorpay refund failed
  • Booking date conflict (table full)
  • User shared location >10 km away (delivery question, edge case)
  • Any error in Meta API 3x in a row
